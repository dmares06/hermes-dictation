# Hermes Hub local workspace

## Goal

Make Hermes useful as a local Wispr Flow replacement by retaining cleaned
dictation history, exposing local usage metrics, and adding lightweight
snippets, Scratchpad notes, and shortcut settings.

## Implementation units

1. Store transcripts, snippets, and notes in a local SQLite database.
2. Save every successful dictation, resolve exact spoken snippets, and support
   fast versus quality Whisper decoding.
3. Serve a loopback-only Hub with Dashboard, History, Snippets, Scratchpad,
   and Settings views.
4. Connect Hub settings and pause controls to the menubar process.
5. Run unit, syntax, API, build, and live Fn-key checks before handoff.

## Product decisions

- The Hub is local-only and does not require an account or network service.
- History stores cleaned text plus raw text, duration, and word count.
- Snippet triggers are exact phrase matches after normalizing final
  punctuation; URL snippets only open `http` and `https` links.
- Quality mode remains the default; Fast mode uses greedy decoding for quicker
  turnaround on longer dictation.
